Resolves #730

## Description
Completes the visual polish for launch by making the theming system production-ready. All hardcoded dark-mode hex colors have been replaced with semantic CSS tokens, ensuring consistent, premium rendering in both light and dark modes.

## Changes
- **Infrastructure**: Activated the full `next-themes` implementation in `ThemeContext.tsx` and exposed the theme toggle in both desktop and mobile navigation.
- **Layout**: Replaced hardcoded `bg-black` with semantic `bg-background text-foreground` in the root layout.
- **Global Polish**: Added `globals.css` overrides for light-mode gradients, smooth transitions, and `hover-lift` shadows.
- **Components**: Replaced all hardcoded hex references (`bg-[#0a0a0a]`, `border-[#262626]`, etc.) with semantic variables (`bg-card`, `border-border`, `bg-muted`) across:
  - Universal components (Navbar, Footer, BrandLogo)
  - UI Library (Card, Modal, Input, FormSelect)
  - Landing modules (Hero, Features, AnimatedStats, CTA, HowItWorks)
  - Global pages (Dashboard, Marketplace, Lending, Tokenize)
  - Sub-components (InvestModal, PoolActionModal, TransactionRow, TransactionHistory)

This ensures the platform respects system preferences, maintains persistent user selections, and aligns the frontend with the premium brand presentation required for the upcoming cycle.

## Verification
- Theme toggle is fully functional.
- Both themes render consistently without invisible text or contrast issues.
- State persists across browser sessions.
